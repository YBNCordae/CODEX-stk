# 校内课时申报与课时费审批 Demo 使用说明

## 1. 这是什么

这是一个用于**学校内部课时量申报、审批和课时费演示**的前端 Demo。

它目前不是正式生产系统，而是一套适合拿去做：

- 客户演示
- 需求沟通
- 业务流程确认
- 后续正式开发前的原型验证

当前版本已经支持两种角色：

- 教师
- 管理员

并且已经具备以下核心功能：

- 教师端独立登录页
- 管理员端独立登录页
- 两端各自注册入口
- 教师填写课时申报
- 系统自动计算折算课时和课时费
- 管理员审批 / 退回
- 管理员批量通过
- 管理员批量导出 CSV

## 2. 先说最重要的：这个 Demo 适合做什么

这套系统现在最适合做的是：

1. 先把“业务流程”跑通。
2. 先让客户看到“系统长什么样、怎么操作”。
3. 先验证“教师怎么填、管理员怎么审、数据怎么导出”。
4. 等客户明确真实规则后，再把课时计算逻辑、字段、权限和数据库接进去。

简单说：

**现在这版适合演示，不适合直接当正式生产系统上线。**

## 3. 当前版本的特点

### 3.1 优点

- 页面完整，流程清晰
- 不依赖后端即可演示
- 本地即可运行
- 可直接 Docker 部署

请务必提前知道这几点：

- 当前数据保存在浏览器本地 `localStorage`

- 当前没有接学校统一认证、数据库、审批权限体系

## 7. 教师端怎么用

教师端主要做一件事：

**自己申报自己所上的课时。**

### 教师操作步骤

1. 打开首页。
2. 点击 `Teacher login`。
3. 用教师账号登录，或者直接注册教师账号。
4. 进入教师工作台后，填写申报表单。
5. 输入以下信息：
   - 学期
   - 课程类型
   - 课程名称
   - 课程代码
   - 教学班
   - 学生人数
   - 授课周数
   - 周学时
   - 额外课时
   - 调节系数
   - 备注
6. 系统会自动在右侧生成：
   - 折算课时
   - 课时费估算
7. 点击提交。
8. 提交后，可以在下方查看自己的申报记录和审批状态。

### 教师能看到哪些状态

- `Pending`
  表示已提交，等待管理员审批
- `Approved`
  表示管理员已通过
- `Returned`
  表示管理员退回，需要修改后重新提交

### 教师被退回后怎么办

如果管理员退回了某条记录：

1. 在教师记录列表中找到该条记录。
2. 点击 `Edit`。
3. 修改内容。
4. 再次提交。

## 8. 管理员端怎么用

管理员端主要做三件事：

1. 看教师提交的数据
2. 审批或退回
3. 批量导出

### 管理员操作步骤

1. 打开首页。
2. 点击 `Admin login`。
3. 用管理员账号登录，或者注册管理员账号。
4. 进入管理员工作台。

### 管理员首页能做什么

- 查看全部申报记录
- 按状态筛选
- 按学期筛选
- 按关键词搜索
- 选择多条记录
- 批量通过
- 导出选中记录
- 导出当前筛选结果

### 单条审批怎么做

1. 在列表中找到某条申报。
2. 点击 `Review`。
3. 右侧会显示：
   - 教师信息
   - 课程信息
   - 折算课时
   - 课时费
   - 计算系数
   - 教师备注
4. 输入审批意见。
5. 点击：
   - `Approve`：审批通过
   - `Return`：退回修改

### 批量审批怎么做

1. 在左侧表格勾选多条记录。
2. 点击 `Approve selected`。
3. 系统会批量通过已勾选且处于待审批状态的记录。

### 批量导出怎么做

管理员端有两种导出方式。

#### 导出选中记录

适合只导出几条指定数据。

步骤：

1. 勾选需要导出的记录。
2. 点击 `Export selected`。

#### 导出当前筛选结果

适合按学期、按状态导出整批数据。

步骤：

1. 先设置筛选条件。
2. 点击 `Export filtered`。

导出文件格式为：

- CSV

## 9. 当前课时计算逻辑

当前系统采用的是一套**通用演示规则**。

### 计算公式

折算课时计算方式：

```text
折算课时 = (周数 x 周学时 + 额外课时) x 课程系数 x 人数系数 x 调节系数
```

课时费计算方式：

```text
课时费 = 折算课时 x 课时单价
```

### 为什么这样设计

因为客户的正式规则目前还没有完全明确，所以先用一套比较通用、容易理解、方便演示的逻辑。

后续如果客户要求：

- 必修课和选修课单价不同
- 不同学院规则不同
- 本科和研究生规则不同
- 理论课和实验课算法不同
- 多级审批

都可以在这套基础上继续改。

## 10. 数据保存在什么地方

当前版本的数据保存在浏览器本地。

也就是说：

- 注册的账号保存在当前浏览器
- 提交的申报记录保存在当前浏览器
- 审批结果也保存在当前浏览器

### 这意味着什么

好处：

- 不需要后端
- 部署简单
- 演示方便

限制：

- 换电脑后数据不会同步
- 换浏览器后数据不会同步
- 清理浏览器缓存后数据可能丢失

这个功能很适合演示前“重置到干净状态”。

## 12. 常见问题

### Q1：为什么我注册的账号在别人电脑上看不到？

因为当前数据保存在本地浏览器，不是保存在服务器数据库里。

### Q2：为什么这套系统现在看起来更像原型？

因为它本来就是用来先跑通流程、先确认需求的 Demo。

### Q3：这套系统能不能直接上线？

技术上可以作为静态演示网站上线，但如果要作为正式系统使用，建议至少补上：

- 后端接口
- 数据库
- 登录认证
- 用户权限
- 正式课时规则
- 操作日志
- 更细的审批流程

### Q4：为什么页面里现在有英文文案？

当前版本为了先保证结构清晰和交互稳定，界面文案以英文为主。后续完全可以统一切换成中文。
